VirtualOrfeo: Testing and Deployment Environment

VirtualOrfeo serves as a digital twin of the ORFEO HPC cluster.

It provides a virtualized environment for safe development and testing.

Digital Twin of ORFEO HPC Cluster

The environment includes a lightweight Kubernetes (K3s) cluster.

This enables efficient container orchestration for applications.

Container Orchestration with Kubernetes

VirtualOrfeo integrates FreeIPA and Authentik for identity and access management.

These tools ensure secure authentication and authorization processes.

Identity and Access Management

NGINX ingress controller and MetalLB load balancer provide external access with TLS certificates issued by FreeIPA CA.

Object Storage and External Access

The environment allows realistic validation of deployment and application behavior.

It ensures authentication and storage integration before production rollout.

Application Architecture and Data Model

The Django application organizes data into a clear hierarchy.

It includes Project, Proposal, Sample, and Experiment levels.

Hierarchy of Data Organization

A Project acts as the top-level container, such as NFFA_DI.

Each Proposal is linked to a Project and owns a dedicated S3 bucket.

Project and Proposal Relationship

Samples belong to Proposals and are identified by slugs.

Experiments belong to Samples and group measurement files.

Sample and Experiment Details

Creation commands ensure validation and transactional safety.

They also handle bucket provisioning and README metadata file writing.

Creation Commands and Validation

The design synchronizes database and storage layers.

It supports consistent provenance tracking.

Data Flow: From Microscopy Lab to ORFEO via the Application

Data acquisition at LAME produces raw TIFF files and metadata captured directly during imaging.

Users upload these large files through a web application using presigned S3 URLs, enabling efficient direct transfer to Ceph RGW without overloading the server.

Data Acquisition at LAME

The app organizes data hierarchically into Projects, Proposals, Samples, and Experiments.

Each is mapped to dedicated S3 buckets and prefixes for structured storage.

Hierarchical Data Organization

Metadata is registered alongside raw data during the upload process.

Background workers asynchronously compute checksums and convert TIFFs into standardized NeXus/NXem files.

Metadata Registration and Processing

Processed NeXus files are stored in mirror buckets.

This ensures FAIR compliance and easy access for downstream analysis.

Storage of Processed Files

Presigned S3 URLs enable efficient direct transfer of large files to Ceph RGW.

This approach prevents server overload and streamlines the upload process.

Efficient Data Transfer

Metadata Handling and NeXus File Construction

Raw TIFF files contain instrument-specific metadata embedded in headers.

This metadata serves as the foundation for subsequent processing and standardization.

Extracting Metadata from Raw TIFF Files

JSON mapping files translate these vendor-specific keys into standardized NeXus/NXem fields.

This enables flexible support for different instruments and ensures compatibility.

Translating Metadata with JSON Mapping Files

The NeXus builder creates a hierarchical HDF5 file with groups for sample, instrument, detector, and measurement events.

These files embed both data and rich metadata for comprehensive representation.

Building Hierarchical NeXus Files

The process includes flattening TIFF metadata, applying mappings, and normalizing start times.

Image data is also embedded to ensure completeness and accuracy.

Processing and Normalizing Metadata

Resulting NeXus files are self-describing, interoperable, and stored alongside raw data.

This ensures adherence to FAIR principles for data management and sharing.

Ensuring FAIR Compliance with NeXus Files

User Interface and Security Model

The web UI features a three-pane management board with HTMX-driven modals for creating proposals, samples, and experiments interactively.

Drag-and-drop file uploads integrate with presigned URLs for efficient data transfer.

Interactive Web UI

Authentication is delegated to Authentik via OIDC, with group claims mapped to Django permissions.

This setup ensures robust role-based access control for users.

Authentication and Role-Based Access Control

Sensitive secrets such as S3 keys and OIDC client secrets are injected securely via Kubernetes Secrets.

This approach enhances the security of critical application credentials.

Secure Secret Management

Presigned URLs enforce time-limited, object-specific access to resources.

This mechanism ensures efficient and secure data handling.

Presigned URLs for Data Access

TLS certificates from FreeIPA CA secure all communications end-to-end.

This guarantees encrypted and trusted interactions across the system.

End-to-End Communication Security

Performance, Scalability, and Future Directions

Uploads and downloads bypass the Django server, flowing directly between browser and object storage.

This approach preserves server resources and enhances efficiency.

Direct Data Flow

Background workers scale independently, processing checksums and NeXus conversions in parallel.

They incorporate retry and idempotency safeguards to ensure reliability.

Independent Background Workers

Database indexing and query optimizations ensure efficient data retrieval.

These measures enhance performance and scalability for large datasets.

Optimized Data Retrieval

Additional features like multi-frame TIFF support and expanded instrument mappings are also in development.

Planned Enhancements

Conclusions and Contributions

This thesis demonstrates a practical, FAIR-by-design data management system for electron microscopy laboratories.

It supports reproducible, scalable, and interoperable data workflows aligned with European open science policies.

FAIR-by-Design Data Management System

A pipeline captures raw data and metadata at acquisition, converting it to the standardized NeXus/NXem format.

This ensures consistency and facilitates interoperability across electron microscopy laboratories.

Standardized Data Pipeline

A web application unifies data upload, annotation, and browsing with secure authentication and authorization.

This tool enhances accessibility and security for managing electron microscopy data.

Unified Web Application

The system integrates with ORFEO’s Ceph-based distributed storage and HPC infrastructure.

A VirtualOrfeo digital twin is used for safe development and deployment validation.

Integration and Validation
